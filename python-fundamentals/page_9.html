Data Types: Tuples and Sets

Tuples: Ordered and Immutable

Tuples store multiple values in a fixed sequence. They are useful when data should not change after creation.

  • Create with parentheses: coords = (10, 20)
  • Access by index: coords[0] gives 10
  • Good for constants, configuration, and safe records.

Tuple Quick Methods

Even without modification methods, tuples still provide useful operations.

  • len(t) counts tuple elements.
  • t.count(x) counts occurrences of a value.
  • t.index(x) returns first position of a value.

Sets: Unique and Unordered

Sets store unique values and automatically remove duplicates.

add()
remove()
discard()
union()
intersection()
difference()

Tuple vs Set in Practice

  • days = ("Mon", "Tue", "Wed") keeps fixed order.
  • tags = {"python", "api", "python"} becomes {"python", "api"}.
  • Use tuples for stable records, sets for uniqueness checks.
  • if "python" in tags: fast membership validation.

Quick Challenge: Create a tuple with 3 cities and a set with repeated categories. Print both and explain why only the set removes duplicates.
